# 📋 LawLaw 8대 실무 시나리오

## 시나리오 1: 초기 상담 및 사건 접수

### 상황
의뢰인이 처음 방문하여 각종 서류를 들고 왔을 때

### 업로드 파일 예시
- 받은 소장 또는 고소장
- 관련 계약서
- 내용증명
- 이메일, 문자 증거
- 기타 참고 자료

### AI 분석 결과
```json
{
  "case_type": "초기 상담",
  "urgency": "높음/보통/낮음",
  "deadline": "답변서 제출 기한: 2024-12-15",
  "initial_assessment": {
    "승소_가능성": "70%",
    "예상_소송비용": "300-500만원",
    "예상_기간": "6-12개월",
    "위험_요소": ["증거 부족", "시효 임박"]
  }
}
```

### 생성 가능한 문서
1. **사건 접수 보고서**
   - 의뢰인 정보
   - 사건 개요 (5W1H)
   - 제출된 증거 목록
   - 초기 법률 검토 의견

2. **법률 검토 의견서**
   - 쟁점 분석
   - 관련 법령 및 판례
   - 승소 가능성 평가
   - 소송 전략 제안

3. **수임료 견적서**
   - 사건 난이도 평가
   - 착수금 / 성공보수
   - 실비 예상액

## 시나리오 2: 민사소송 - 원고 대리 (돈 받아야 하는 경우)

### 상황
의뢰인이 돈을 받지 못했거나, 계약 위반으로 손해를 입은 경우

### 업로드 파일 예시
- 계약서
- 거래 내역서 (입금 증빙)
- 채무 인정 문자/이메일
- 내용증명 발송 증명
- 담보 관련 서류

### AI 분석 결과
```json
{
  "case_type": "민사소송_원고",
  "claim_type": "채권 청구",
  "claim_amount": "50,000,000원",
  "legal_grounds": [
    "민법 제390조 (채무불이행)",
    "상법 제48조 (상행위로 인한 채무)"
  ],
  "statute_of_limitations": {
    "type": "상사채권",
    "period": "5년",
    "deadline": "2026-03-15",
    "status": "안전"
  }
}
```

### 생성 가능한 문서
1. **소장 (대여금/물품대금/손해배상 청구)**
   - 당사자 표시
   - 청구 취지: "피고는 원고에게 금 5,000만원 및 이에 대한 지연손해금을 지급하라"
   - 청구 원인:
     - 계약 체결 경위
     - 이행 내역
     - 불이행 사실
     - 손해 발생

2. **준비서면 (원고)**
   - 피고 주장에 대한 반박
   - 추가 증거 제출
   - 법리 보충

3. **증거 목록**
   - 갑 제1호증: 거래 계약서
   - 갑 제2호증: 입금 확인증
   - 갑 제3호증: 내용증명

4. **답변서 예상 및 대응 전략**
   - 피고가 할 만한 주장 예측
   - 선제적 대응 방안

## 시나리오 3: 민사소송 - 피고 대리 (돈 달라는 소송 당한 경우)

### 상황
의뢰인이 소장을 받았고, 억울하거나 금액에 이의가 있는 경우

### 업로드 파일 예시
- 받은 소장
- 계약서 (원고가 제출한 것과 다를 수 있음)
- 이미 갚은 증거 (입금 확인증)
- 반박 자료 (하자 증명, 계약 무효 사유)

### AI 분석 결과
```json
{
  "case_type": "민사소송_피고",
  "plaintiff_claim": "5,000만원 청구",
  "defense_strategy": [
    "일부 인정 (2,000만원은 맞음)",
    "일부 부인 (3,000만원은 과다 청구)",
    "상계 항변 (오히려 우리가 받을 돈 1,000만원)"
  ],
  "counterclaim_possible": true,
  "deadline": "답변서 제출: 2024-12-15 (14일 후)"
}
```

### 생성 가능한 문서
1. **답변서**
   - 원고 주장에 대한 인정/부인
   - 항변 사유 (소멸시효, 상계 등)
   - 예비적 주장

2. **준비서면 (피고)**
   - 원고 증거에 대한 반박
   - 피고 측 유리한 사정 부각

3. **반소장** (필요시)
   - 오히려 원고가 우리에게 돈을 줘야 하는 경우

4. **증거 목록**
   - 을 제1호증: 입금 확인증 (이미 갚은 증거)
   - 을 제2호증: 하자 사진

## 시나리오 4: 형사변호 - 피고인 변호 (형사 기소당한 경우)

### 상황
의뢰인이 형사 고소/고발당했거나 기소된 경우

### 업로드 파일 예시
- 기소장 또는 약식 명령
- 경찰 조서
- 고소장 사본
- 증거 자료 (CCTV, 녹취록 등)

```json
{

  "prosecution_evidence": [
    "피해자 진술",
    "거래 내역서",
    "계약서"
  ],
  "defense_direction": "무죄 주장",
  "fallback_strategy": "정상 참작 (감형)",
  "sentencing_range": {
    "법정형": "10년 이하 징역",
    "예상_구형": "징역 3년",
    "예상_선고": "징역 2년 집행유예 3년"
  }
}
```

### 생성 가능한 문서
1. **변론 요지서**
   - 무죄 주장 근거
   - 검사 증거 탄핵
   - 유리한 정황 강조

2. **의견서 (검찰 제출)**
   - 불기소 또는 약식 처분 요청
   - 피의자 유리한 사정

3. **양형 참고자료**
   - 반성문
   - 피해 회복 노력
   - 탄원서 (가족, 직장)
   - 유사 사건 판례 (집행유예 선고)

4. **항소 이유서** (1심 패소 시)
   - 사실 오인
   - 법리 오해
   - 양형 부당

- 피해 증거 (계약서, 거래 내역)
- 문자/이메일 (사기 정황)
- 피의자 신상 정보
- 고소 의사 확인서

  "prosecution_likelihood": "80%",
  "civil_claim": {
    "가능": true,
    "방법": "배상 명령 신청 또는 별도 민사소송"

### 생성 가능한 문서

   - 경찰/검찰이 수사 미온적일 때

3. **의견서 (수사 단계)**
   - 피의자 허위 진술 반박
   - 추가 증거 제출

## 시나리오 7: 내용증명/경고장

### 상황
소송 전 단계에서 상대방에게 의사를 명확히 전달하고 증거를 남기고 싶은 경우

### 업로드 파일 예시
- 채권 증빙 (차용증, 계약서)
- 위반 사실 증거
- 이전 독촉 내역

### AI 분석 결과
```json

  "legal_effect": [

    "소송 전 증거 확보",

  ],
  "next_step": "7일 내 미이행 시 소송 제기"

```

### 생성 가능한 문서
1. **내용증명**
   - 제목: "채무 이행 최고의 건"

     - 채권 발생 경위
     - 현재 미지급액
     - 이행 기한 (7일)

2. **경고장**
   - 계약 위반 중지 요구
   - 손해배상 청구 예고

3. **합의서** (상대방이 응답 시)

   - 합의 조항

## 🎯 시나리오 자동 감지 로직

```python
def detect_scenario(uploaded_files, analysis_result):
    """
    업로드된 파일과 AI 분석 결과를 바탕으로 시나리오 판단
    """

    # 키워드 기반 시나리오 매칭
    if "소장" in files or "피고" in analysis:
        if "원고" in analysis:
            return "시나리오2_민사원고"
        else:
            return "시나리오3_민사피고"

    elif "기소장" in files or "형사" in analysis:
        return "시나리오4_형사변호"

    elif "사기" in analysis or "횡령" in analysis:
        return "시나리오5_형사고소"

    elif "계약서" in files and not ("소장" in files):
        return "시나리오6_계약검토"

    elif "채권" in analysis or "독촉" in analysis:
        return "시나리오7_내용증명"

    elif "사고" in files or "진단서" in files:
        return "시나리오8_손해배상"

    else:
        return "시나리오1_초기상담"
```
